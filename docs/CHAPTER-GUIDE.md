# Chapter Setup Guide

This is the full walkthrough for running APL at your GDG chapter. Estimated setup time: **2–3 hours** end-to-end (most of it Cloudflare DNS propagation).

## 1. Prerequisites

- A **Cloudflare** account with a domain (free tier is enough — Workers + Pages + DNS all included).
- A **GitHub** account / organisation for the fork.
- Node 20+, Python 3.11+, `git`, `gh` (GitHub CLI), `wrangler` (`npx wrangler` works).
- (Post-event) Claude Code CLI for the AI vetting pipeline — _or_ you can do scoring manually using the criteria in `pipeline/VETTING-INSTRUCTIONS.md`.

## 2. Fork & rename

```bash
# Fork the repo on GitHub first, then:
git clone git@github.com:<your-org>/<your-repo>.git
cd <your-repo>
```

Update:
- `event.config.json` → `github.owner` and `github.repo` to match your fork.
- `host/electron/main.cjs` → `REPO` constant (used for auto-update of the projector app).
- `host/package.json` → `build.publish.repo` (electron-builder release target).

## 3. Edit `event.config.json`

This is the **only** file you need to edit for chapter-specific values. Everything downstream reads from it.

```jsonc
{
  "chapter": { "name": "GDG <City>", "city": "<City>" },
  "event": {
    "title": "<Event Title>",
    "year": 2027,
    "window": {
      "start": "2027-MM-DDTHH:MM:00",   // event start
      "end":   "2027-MM-DDTHH:MM:00",   // hard deadline for code commits
      "timezone": "Asia/Kolkata"        // affects rollback cutoff
    }
  },
  "challenges": [ { "id": "...", "color": "blue", "title": "...", "prompt": "..." } ],
  "urls": {
    "web":  "https://apl.<your-domain>",
    "api":  "https://apl-api.<your-domain>",
    "form": "https://forms.gle/<your-form>",   // generated in step 7
    "repo": "https://github.com/<your-org>/<your-repo>"
  },
  "data_dir": "data/<your-chapter-and-year>",
  "featured_quotes": []                          // fill in after the event
}
```

Brand colors and font default to Google's; override in `web/styles.css` and `web/preview/`-generated files if you want a different palette.

## 4. Cloudflare: Worker + Pages

```bash
# Set up auth once:
export CLOUDFLARE_API_TOKEN=<your-token-with-workers+pages+dns-permissions>

# 4a. Deploy the Worker (reactions / WS fan-out):
cd api
# Edit wrangler.toml: account_id + ALLOWED_ORIGIN should match your web URL
# Generate a host secret + set it:
HOST_SECRET=$(openssl rand -hex 24)
echo "Save this somewhere safe: $HOST_SECRET"
echo "$HOST_SECRET" | npx wrangler secret put HOST_SECRET
npx wrangler deploy

# 4b. Add a custom domain on the Worker:
# Either in wrangler.toml (uncomment the routes block) and redeploy,
# or via dashboard: Workers > apl-api > Custom Domains > add apl-api.<your-domain>

# 4c. Create a Pages project + first deploy:
cd ../
npx wrangler pages project create apl-web --production-branch=main
npx wrangler pages deploy ./web --project-name=apl-web --branch=main

# 4d. Attach apl.<your-domain> to the Pages project (dashboard or API).
# Add a CNAME apl → apl-web.pages.dev if Cloudflare doesn't auto-create it.

# 4e. (Optional but recommended) If your zone has a non-zero Browser Cache TTL:
# add a Cache Rule scoped to apl.<your-domain> with browser_ttl=respect_origin.
# See contestants/ → README in this repo's git history for the API call.
```

## 5. Web: customise

The web app is plain HTML/CSS/JS — no build step. Files of interest:

- `web/index.html` and `web/styles.css` / `web/app.js` — homepage (re-generated by `pipeline/build-preview.py` after the event; before, it's a simple intro page you write yourself).
- `web/scoreboard/` — generated by `pipeline/build-scoreboard.py` after scoring. Pre-event you can leave this empty.

Before the event you probably want a simple landing page with the challenge briefs + the form URL. Edit `web/index.html` directly until you have judging results to publish.

## 6. Host: build the projector app

```bash
cd host
npm install
# Bake config (writes electron/build-config.js with the host secret + QR data URLs):
export APL_HOST_SECRET=<the-secret-from-step-4a>
npm run bake

# Local test:
npm run dev    # runs Vite + Electron together

# Production builds:
APL_HOST_SECRET=... npm run dist:dmg    # mac
APL_HOST_SECRET=... npm run dist:win    # windows
APL_HOST_SECRET=... npm run dist:linux  # appimage + deb

# To cut a tagged release (mac/win/linux + GitHub release):
APL_HOST_SECRET=... bash scripts/release.sh 0.1.0
```

Shortcuts in the running app:
- `F11` — fullscreen
- `F2` — toggle the QR sticker (hide when you don't want it covering the corner)
- `F3` — show the submit-results page (fullscreen form QR)

## 7. Day of event

1. Make a Google Form for submissions (fields in `pipeline/FORM-FIELDS.md` — port over the template). Note the `forms.gle/...` short link.
2. Update `event.config.json` → `urls.form` to point at it.
3. Rebuild + redeploy the host so the F3 QR points at the right form. (`npm run bake && bash scripts/release.sh <next-version>`)
4. On the projector machine, install the host app, sign into Hotstar (or whatever stream), press Space to dismiss the poster, `F11` to fullscreen.
5. During the event, attendees scan the top-left QR → land on `apl.<your-domain>` → tap emoji → reactions fly across the projector.
6. At deadline, press `F3` to switch to the submit-results screen. Tell attendees to scan it.

## 8. Post-event: scoring pipeline

Once the form has responses, run the pipeline from `pipeline/`:

```bash
cd pipeline

# 8a. Export the form responses to .xlsx, save the file path
# 8b. Edit clone-all.py to point at the xlsx (or use the helper xlsx→json script)
# 8c. Pull repos:
python3 clone-all.py

# 8d. Unshallow (so we get full commit history)
# Loop git fetch --unshallow over pipeline/<slug>/ — script in repo: TBD

# 8e. Analyse commit timing:
python3 check-commit-timing.py

# 8f. Compute rollback plan (which repos need to be rewound to deadline state):
python3 compute-rollback.py
python3 do-rollback.py

# 8g. Vetting:
#   - Either use Claude Code agents (see VETTING-INSTRUCTIONS.md) — split records
#     into batch-1..5.json and dispatch parallel agents writing scores-batch-N.json.
#   - Or do it manually using the rubric (1-10 on agentic / demo / quality / fit / originality).
#
#   For repos that needed a rollback, run a SECOND vetter on the rolled-back state
#   and write scores-rollback.json.

# 8h. Finalise the ranking (floor scores for empty-at-deadline, prefer re-vet for needs-rollback):
python3 finalize-ranking.py

# 8i. Render the scoreboard + homepage:
python3 build-scoreboard.py
python3 build-preview.py

# 8j. Deploy:
cd ..
npx wrangler pages deploy ./web --project-name=apl-web --branch=main --commit-dirty=true
```

The published `/scoreboard` shows each project's 5-axis score, judge reasoning, commit-timeline, and direct repo/demo links. The homepage shows the top-3 podium, the activity heartbeat, award highlights, the stack tag-cloud, and quotable one-liners pulled from your `event.config.json` `featured_quotes`.

## Privacy notes

- `pipeline/submissions.json` (full form export with emails/phone) is in `.gitignore` — keep it local.
- `pipeline/<slug>/` (cloned repos) are in `.gitignore` — they're not yours to ship.
- `data/<chapter-year>/` contains scores + ranking JSONs without raw PII (just first names + project info as it appears on the public scoreboard).
- The form has a *"Can we share your submission publicly?"* field; `build-scoreboard.py` filters out anyone who answered "No".

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `apl.<domain>` shows the Pages 404 page | DNS not propagated, or custom domain not attached | Wait ~5 min after attaching; check the CNAME record |
| Reactions don't show on projector | WS disconnected after Worker deploy | `Cmd+R` the projector window — auto-reconnect kicks in within a minute on 0.3.6+ |
| Everyone's reactions stop after a burst | Per-IP rate limit (whole venue shares one wifi IP) | Bump `IpRateLimiter(N, M)` in `api/src/room.ts` to higher numbers and redeploy |
| Builder's git log shows 2025-01-01 commits | Probably a template scaffold (Lovable, Bolt, TanStack starter, etc.) | `check-commit-timing.py` automatically classifies these as `TEMPLATE_DERIVED` — no penalty |
